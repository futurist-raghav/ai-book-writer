## New Components

### 1. **GanttChart.tsx** (~350 lines)

**Purpose:** Visualize events as timeline bars with duration-based positioning

**Features:**
- Canvas-based rendering for performance
- Timeline position vs. duration mapping
- Status-based color coding
- Dependency arrows between events
- Zoom (scroll wheel) and pan controls
- Interactive event selection
- Event type labels on bars
- Progress indicator for in-progress events

**Props:**
```typescript
interface GanttChartProps {
  events: FlowEvent[];
  dependencies?: FlowDependency[];
  onEventClick?: (eventId: string) => void;
  isLoading?: boolean;
  error?: string;
}
```

**Key Features:**
- Horizontal timeline with grid lines
- Bar width = duration × scale
- Dependency arrows show relationships
- Status legend with event counts
- Zoom controls (10px to 200px scale)
- Hover highlighting for events

**Use Case:** Project managers see overall timeline and dependencies at a glance

### 2. **FlowFilter.tsx** (~280 lines)

**Purpose:** Multi-criteria filtering for events

**Features:**
- Search by title/description
- Filter by event type (act, scene, beat, etc.)
- Filter by status (planned, in_progress, completed, blocked)
- Date range filtering (by timeline position)
- Active filter count badge
- Clear all / reset functionality
- Collapsible UI
- Filter state export

**Exports:**
```typescript
interface FlowFilterState {
  types: string[];
  statuses: string[];
  dateRange: { start?: string; end?: string };
  searchQuery: string;
}

function FlowFilter(props: FlowFilterProps): JSX.Element
function applyFilters(events: FlowEvent[], filters: FlowFilterState): FlowEvent[]
```

**Key Features:**
- Checkbox-based type/status selection
- Number inputs for timeline range
- Search debouncing
- Visual active filter indicators
- Reusable filter logic

**Use Case:** Find specific events based on type, status, or location in timeline

### 3. **BulkOperations.tsx** (~380 lines)

**Purpose:** Multi-select events and perform batch operations

**Features:**
- Checkbox select all / individual events
- Status bar showing selection count
- Batch status change dropdown
- CSV export of selected events
- Batch delete with confirmation dialog
- Individual event deselection from preview
- Indeterminate state for partial selection
- Disabled state during operations

**Props:**
```typescript
interface BulkOperationsProps {
  events: FlowEvent[];
  onBatchStatusChange?: (eventIds: string[], newStatus: string) => void;
  onBatchDelete?: (eventIds: string[]) => void;
  onBatchUpdate?: (eventIds: string[], data: Partial<FlowEvent>) => void;
  isLoading?: boolean;
}
```

**Key Features:**
- Full event list with checkboxes
- Selection preview (shows first 5, +N more)
- CSV export with headers
- Confirmation modal before delete
- Warning about cascading soft-deletes
- Quick status dropdown
- Clear selection button

**Use Case:** Update many events at once or export data

### 5. **EnhancedFlowDashboard.tsx** (~250 lines)

**Purpose:** Main orchestration component for P2.3 features

**Features:**
- 4 view modes: Overview, Gantt, Analytics, Bulk Ops
- Integrated filter component
- Filtered event/dependency lists
- Unified event handling
- View mode toggle buttons
- Active filter indicator

**Props:**
```typescript
interface EnhancedFlowDashboardProps {
  bookId: string;
  events: FlowEvent[];
  dependencies: FlowDependency[];
  onEventCreate?: (event: Partial<FlowEvent>) => void;
  onEventUpdate?: (eventId: string, data: Partial<FlowEvent>) => void;
  onBatchStatusChange?: (eventIds: string[], newStatus: string) => void;
  onBatchDelete?: (eventIds: string[]) => void;
  onDependencyCreate?: (fromId: string, toId: string) => void;
  onDependencyDelete?: (depId: string) => void;
  isLoading?: boolean;
  error?: string;
}
```

**View Modes:**
1. **Overview** - P2.2 FlowDashboard (Timeline, Graph, Editor)
2. **Gantt** - Horizontal timeline bars with duration
3. **Analytics** - Metrics and burn-down charts
4. **Bulk Ops** - Multi-select and batch operations

## Technical Specifications

### Canvas Rendering (GanttChart)
- Grid lines every 10 units
- Bar height: 30px per event
- Row height: 60px (includes spacing)
- Zoom: 10-200px per unit
- Scroll: horizontal pan support
- Performance: Optimized for 100+ events

### Filtering (FlowFilter)
- Types: 6 event types supported
- Statuses: 4 statuses supported
- Date Range: Number input (timeline_position)
- Search: String match on title + description
- Case-insensitive search

### Bulk Operations (BulkOperations)
- Max simultaneous selection: unlimited
- CSV headers: Title, Type, Status, Position, Duration
- Batch size: No limit (backend handles pagination)
- Confirmation gate: Delete operations only

### Analytics (FlowAnalytics)
- Completion formula: (completed / total) × 100
- Burn rate: (completion_rate / time_elapsed)
- Avg duration: (sum of durations) / count
- Burn-down: Linear projection

## Performance Considerations

### Client-Side Operations
- Filtering: O(n) per keystroke - debounce recommended
- Sorting: O(n log n) for initial sort
- Bulk operations: O(n) selection management

### Canvas Rendering (GanttChart)
- Redraws on: zoom, scroll, hover
- Batches: Draw calls optimized
- Memory: Efficient for 1000+ events

### Recommendations
- Virtualization for event lists >500 items
- Request debouncing for filter changes
- Pagination for bulk export (>1000 events)
