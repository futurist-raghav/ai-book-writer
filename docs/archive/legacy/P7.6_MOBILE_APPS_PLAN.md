## рџЏ—пёЏ TECHNICAL ARCHITECTURE

### Tech Stack
- **Framework:** React Native (Expo for rapid deployment)
- **State:** TanStack Query + Context API
- **Storage:** WatermelonDB (Nozbe) for offline-first
- **Authentication:** OAuth 2.0 (existing endpoints)
- **API:** REST (existing FastAPI backend)
- **Building:** Expo EAS Build for iOS/Android

### Data Sync Strategy
```
Mobile в†ђ в†’ Backend (REST API)
  в†“
WatermelonDB (Local Cache)
  в†“
Action Queue (Offline Writes)
```

1. **On Launch:**
   - Check connectivity
   - If online: Fetch latest user data (books, chapters, settings)
   - If offline: Load from WatermelonDB cache

2. **Chapter Fetch:**
   - Try backend first (if online)
   - Fall back to WatermelonDB
   - Cache all fetches locally

3. **Chapter Draft:**
   - Save immediately to WatermelonDB
   - Queue sync action
   - Sync on next connection

4. **Sync:**
   - On reconnect, process action queue
   - Batch API calls (POST /chapters, PATCH /chapters/{id})
   - Handle conflicts (show user summary)
   - Persist success/failure state
