## Error Scenarios to Test

### Authorization Errors
```bash
# No token
curl -X GET "http://localhost:8000/api/v1/books/{book_id}/events"
# Expected: 401 Unauthorized

# Invalid token
curl -X GET "http://localhost:8000/api/v1/books/{book_id}/events" \
  -H "Authorization: Bearer invalid_token"
# Expected: 401 Unauthorized or 403 Forbidden

# Another user's book
curl -X GET "http://localhost:8000/api/v1/books/{another_users_book_id}/events" \
  -H "Authorization: Bearer ${OTHER_USER_TOKEN}"
# Expected: 403 Forbidden
```

### Validation Errors
```bash
# Invalid event_type
curl -X POST "http://localhost:8000/api/v1/books/{book_id}/events" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"title": "Test", "event_type": "invalid_type"}'
# Expected: 422 Unprocessable Entity

# Missing required field
curl -X POST "http://localhost:8000/api/v1/books/{book_id}/events" \
  -H "Authorization: Bearer ${TOKEN}" \
  -d '{"description": "No title"}'
# Expected: 422 Unprocessable Entity
```

### Resource Not Found
```bash
# Non-existent event
curl -X GET "http://localhost:8000/api/v1/books/{book_id}/events/00000000-0000-0000-0000-000000000000" \
  -H "Authorization: Bearer ${TOKEN}"
# Expected: 404 Not Found

# Non-existent book
curl -X GET "http://localhost:8000/api/v1/books/00000000-0000-0000-0000-000000000000/events" \
  -H "Authorization: Bearer ${TOKEN}"
# Expected: 403 Forbidden (user doesn't own it)
```

### Dependency Cycle Prevention
```bash
# Create self-dependency
curl -X POST "http://localhost:8000/api/v1/books/{book_id}/events/{event_id}/dependencies" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"to_event_id": "{same_event_id}", "dependency_type": "blocks"}'
# Expected: 400 Bad Request (Cannot create self-dependency)
```

## Troubleshooting

### Migration Won't Apply
```bash
# Check migration status
docker compose exec backend alembic current

# If stuck, downgrade and retry
docker compose exec backend alembic downgrade 009_entity_references
docker compose exec backend alembic upgrade head
```

### Import Errors
```bash
# Run validation script
python validate_flow_engine.py

# This will show exactly which imports fail and why
```

### Test Failures
```bash
# Run specific test with verbose output
docker compose exec backend pytest tests/test_flow_engine.py::TestFlowEngineAPI::test_create_flow_event -vv

# Shows exact failure point and assertion details
```
