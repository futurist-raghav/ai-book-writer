## Message Protocol

### Client → Server
```typescript
// Cursor movement
{ type: 'cursor_move', position: number, selection?: { from, to } }

// Text change
{ type: 'text_edit', delta: object }

// User actions
{ type: 'comment_added', data: commentData }
{ type: 'suggestion_added', data: suggestionData }
{ type: 'typing', is_typing: boolean }

// Sync request
{ type: 'request_sync' }
{ type: 'ping' }
```

### Server → All Clients
```typescript
// Presence updates
{ type: 'presence_update', users: PresenceUser[] }

// Cursor updates
{ type: 'cursor_update', user_id, position, selection }

// Text edits
{ type: 'text_edit', user_id, delta, timestamp }

// Notifications
{ type: 'comment_added', data, timestamp }
{ type: 'suggestion_added', data, timestamp }
{ type: 'notification', data, timestamp }

// Individual client
{ type: 'user_typing', user_id, is_typing }
{ type: 'sync_response', content, compiled_content, last_modified }
{ type: 'pong' }
```

## Usage in Components

### Basic Setup
```typescript
const {
  isConnected,
  presenceUsers,
  sendCursorMove,
  sendTextEdit,
  sendCommentAdded,
  sendSuggestionAdded,
  sendTypingIndicator,
} = useRealtimeCollaboration({
  chapterId: 'chapter-id',
  token: authToken,
  onPresenceUpdate: (users) => setUsers(users),
  onUserTyping: (userId, isTyping) => updateUI(),
});

// In render
<CollaborationPresenceBar
  users={presenceUsers}
  currentUserId={userId}
  isConnected={isConnected}
  typingUsers={typingUsers}
/>
```

### Sending Events
```typescript
// User moved cursor
sendCursorMove(42, { from: 40, to: 44 });

// User edited text
sendTextEdit({ ops: [{ retain: 42 }, { insert: 'new' }] });

// User added comment
sendCommentAdded({ id: 'c1', text: 'review this', author: u1 });

// Show typing
sendTypingIndicator(true);
```

## Architecture

```
Browser (Client A)
├─ WriterCanvas
│  ├─ useRealtimeCollaboration → WebSocket {
│  │                              type: 'cursor_move'
│  │                              position: 42
│  │                            }
│  └─ CollaborationPresenceBar
│     └─ Avatar stack (Jane, Bob, Carol)
│
[WebSocket Server @ /api/v1/ws/chapters/{id}/collaborate]
│
├─ ConnectionManager.active_connections['ch-1'] = {
│  'user-a': <socket_a>,
│  'user-b': <socket_b>,
│  'user-c': <socket_c>
│}
│
├─ OnMessage from user-a
│  └─ Broadcast to user-b, user-c (exclude sender)
│
└─ Broadcast message
   ├─ Browser (Client B) receives 'cursor_update'
   └─ Browser (Client C) receives 'cursor_update'
```

## Performance Notes

- **Heartbeat:** 30 seconds (keeps connection alive)
- **Message types:** Discriminated union (prevents typos in message protocol)
- **Dead connections:** Cleaned up automatically on send failure
- **Presence tracking:** In-memory Set (O(1) lookups)
- **Typing indicators:** Debounced at client level (no spam)

## Error Handling

- WebSocket close/error → Auto-reconnect (up to browser limits)
- Message parse error → Console warning, connection continues
- Dead socket in broadcast → Removed from connection pool
- Auth token invalid → Connection rejected by server
- Chapter not found → 404 on REST endpoints (WebSocket auth handled separately)
