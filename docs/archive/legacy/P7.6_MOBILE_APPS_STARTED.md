#### 2. **OfflineDraftService**
```python
# app/services/mobile.py

@staticmethod
def create_draft(db: Session, user_id: str, data: OfflineDraftCreate) -> OfflineDraft
  # Create new offline draft
  # Query: chapter to validate and get book_id
  # Set sync_status = "pending", local_version = 1
  # Return draft

@staticmethod
def sync_draft(db: Session, user_id: str, data: OfflineDraftSync) -> OfflineDraft
  # Sync client draft to server
  # Query: existing draft on (user_id, chapter_id, device_id)
  # If exists: compare local_version vs synced version for conflict detection
  #   - If no conflict: update content + increment synced_version, set sync_status = "synced"
  #   - If conflict: set sync_status = "conflicted" (client shows merge UI)
  # If not exists: create new draft with status = "synced"
  # Return draft

@staticmethod
def get_pending_drafts(db: Session, user_id: str) -> List[OfflineDraft]
  # Get all drafts with sync_status = "pending" or "conflicted"
  # Used by app to show queue during offline reconnect
  # Return list

@staticmethod
def resolve_conflict(db: Session, user_id: str, draft_id: str, content: str) -> OfflineDraft
  # User manually resolves conflict
  # Update draft content + sync_status = "synced"
  # Return draft

@staticmethod
def delete_draft(db: Session, user_id: str, draft_id: str) -> bool
  # User discards offline draft
  # Soft delete (mark synced_at with null + status = "discarded")
  # Return success
```

**Conflict Resolution Logic:**
```python
# In sync_draft():
if draft.sync_status == "synced":
    # Server has latest version (synced_at is set)
    if data.local_version > draft.local_version:
        # Local version ahead: safe to sync
        draft.content = data.content
        draft.sync_status = "synced"
    elif data.local_version == draft.local_version:
        # Same version: update allowed
        draft.content = data.content
    else:
        # Local version behind: potential conflict
        # Mark as conflicted, client shows merge dialog
        draft.sync_status = "conflicted"
```

### API Routes (12+)

All routes use `get_current_user` auth dependency. All user_id filters for isolation.

#### Sessions

```python
POST   /api/v1/mobile/sessions
  Body: { device_id, device_type, app_version }
  Response: { id, device_id, is_active, created_at }
  Action: Create/update session, log device info

POST   /api/v1/mobile/sessions/{device_id}/activity
  Action: Update last_activity_at (heartbeat)
  Response: { last_activity_at }

POST   /api/v1/mobile/sessions/{device_id}/push-token
  Body: { push_token }
  Action: Store FCM or APNs token
  Response: { status: "token_updated" }
```

#### Offline Drafts

```python
POST   /api/v1/mobile/drafts
  Body: { chapter_id, content, device_id }
  Response: { id, chapter_id, sync_status, local_version }
  Action: Create offline draft

POST   /api/v1/mobile/drafts/sync
  Body: { chapter_id, content, local_version, device_id }
  Response: { id, sync_status, synced_at }
  Action: Sync draft + conflict detection

GET    /api/v1/mobile/drafts/pending
  Response: [{ id, chapter_id, book_id, sync_status, local_version, updated_at }]
  Action: List pending/conflicted drafts for sync queue UI
```

#### Notifications

```python
GET    /api/v1/mobile/notifications
  Response: [{ id, title, body, is_read, deep_link, created_at }]
  Action: Get 50 recent notifications

GET    /api/v1/mobile/notifications/unread-count
  Response: { unread_count: int }
  Action: Get badge count (cheap query)

POST   /api/v1/mobile/notifications/{notification_id}/read
  Response: { read_at }
  Action: Mark notification as tapped
```

#### Voice Notes

```python
POST   /api/v1/mobile/voice-notes
  Body: { book_id, chapter_id?, audio_url, audio_duration_seconds, language }
  Response: { id, transcription_status, language }
  Action: Create voice note, queue for transcription

GET    /api/v1/mobile/voice-notes/{note_id}
  Response: { id, transcription, transcription_status, transcription_confidence, is_converted_to_text }
  Action: Get note + transcription (when ready)

POST   /api/v1/mobile/voice-notes/{note_id}/convert

  Action: Insert transcription into chapter
```

#### Reading Mode
