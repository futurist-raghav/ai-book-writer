# P7.1 Writing Performance Tools - Implementation Guide

## Overview

P7.1 Writing Performance Tools provides writers with comprehensive tracking of their writing productivity, motivation, and achievement. This phase implements session tracking, milestones, streaks, and writing challenges.

## Features Implemented

### 1. Writing Sessions
- **Track individual writing sessions** with start/end times
- **Record metrics**: words written, words deleted, net words, characters changed
- **Session types**: focused, editing, brainstorm
- **Optional notes** for session context

**API Endpoints:**
- `POST /api/v1/writing/sessions` - Start new session
- `PATCH /api/v1/writing/sessions/{id}` - End session and record metrics
- `GET /api/v1/writing/stats` - Get aggregated statistics

### 2. Writing Streaks
- **Automatic streak tracking** - increments based on daily writing
- **Current and longest streaks** - motivational tracking
- **Streak recovery** - continue from yesterday or start new

**Automatic Updates:**
- Updated whenever a session ends
- Checks calendar dates to maintain accuracy
- Tracks total days written

### 3. Writer Milestones
- **Predefined milestones**:
  - 1,000 words written
  - 5,000 words written
  - 10,000 words written
  - 50,000 words written
- **Progress tracking** - percentage towards milestone
- **Unlock notifications** - when goals are achieved

**Automatic Updates:**
- Checked after each session
- Progress percent calculated automatically
- Unlocked status and timestamp recorded

### 4. Writing Challenges
- **Create custom challenges** - set goals and deadlines
- **Challenge types**: 30-day challenge, NaNoWriMo, custom
- **Progress tracking** - real-time word count updates
- **Completion tracking** - marks complete when target reached

**API Endpoints:**
- `POST /api/v1/writing/challenges` - Create challenge
- `PATCH /api/v1/writing/challenges/{id}` - Update progress
- Challenges embedded in stats endpoint

### 5. Aggregated Statistics
- **Total words written** across all sessions
- **Session metrics** - count, average length
- **Streak data** - current and longest
- **Challenge status** - count of active challenges
- **Milestones** - count of unlocked achievements
- **Last session** - timestamp for resumption

## Database Schema

### writing_sessions
```sql
CREATE TABLE writing_sessions (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) FOREIGN KEY,
  book_id VARCHAR(36) FOREIGN KEY (nullable),
  started_at DATETIME,
  ended_at DATETIME (nullable),
  words_written INT DEFAULT 0,
  words_deleted INT DEFAULT 0,
  net_words INT DEFAULT 0,
  characters_changed INT DEFAULT 0,
  session_type VARCHAR(50),
  notes VARCHAR(255),
  created_at DATETIME DEFAULT NOW()
);

CREATE INDEX ix_writing_sessions_user_id ON writing_sessions(user_id);
CREATE INDEX ix_writing_sessions_started_at ON writing_sessions(started_at);
```

### writing_streaks
```sql
CREATE TABLE writing_streaks (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) UNIQUE FOREIGN KEY,
  current_streak INT DEFAULT 0,
  longest_streak INT DEFAULT 0,
  last_written_at DATETIME,
  total_days_written INT DEFAULT 0,
  total_words_written INT DEFAULT 0,
  updated_at DATETIME DEFAULT NOW(),
  created_at DATETIME DEFAULT NOW()
);
```

### writer_milestones
```sql
CREATE TABLE writer_milestones (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) FOREIGN KEY,
  book_id VARCHAR(36) FOREIGN KEY (nullable),
  milestone_type VARCHAR(50),
  title VARCHAR(255),
  description VARCHAR(500),
  target_value INT,
  current_value INT DEFAULT 0,
  progress_percent INT DEFAULT 0,
  is_unlocked INT DEFAULT 0,
  unlocked_at DATETIME (nullable),
  created_at DATETIME DEFAULT NOW()
);
```

### writing_challenges
```sql
CREATE TABLE writing_challenges (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) FOREIGN KEY,
  book_id VARCHAR(36) FOREIGN KEY (nullable),
  challenge_name VARCHAR(255),
  challenge_type VARCHAR(50),
  target_words INT,
  current_words INT DEFAULT 0,
  progress_percent INT DEFAULT 0,
  started_at DATETIME,
  ends_at DATETIME,
  completed_at DATETIME (nullable),
  is_active INT DEFAULT 1,
  is_completed INT DEFAULT 0,
  created_at DATETIME DEFAULT NOW()
);
```

## Integration Points

### With Existing Features
- **Books**: Sessions can be book-scoped via `book_id`
- **Users**: All data is user-isolated via `user_id` FK
- **Auth**: Routes require `get_current_user` dependency
- **Dashboard**: Stats can be displayed in writer profile

### Data Flow
1. Writer clicks "Start Session" → `POST /writing/sessions`
2. Backend creates session record with current timestamp
3. Frontend tracks time and word delta locally (or via input)
4. On "End Session" → `PATCH /writing/sessions/{id}` with metrics
5. Backend updates session record, triggers:
   - Streak update (check last write date)
   - Milestone check (evaluate progress)
   - Challenge progress (if book-scoped)
6. Frontend calls `GET /writing/stats` to refresh dashboard

## Known Limitations

1. **Streak Logic**: Currently day-based; sub-day streaks not supported
2. **Metrics**: Relies on manual entry of word counts (not live text analysis)
3. **Challenges**: Single-user only; no multi-writer challenges
4. **Milestones**: Pre-defined list; custom milestones not supported yet
