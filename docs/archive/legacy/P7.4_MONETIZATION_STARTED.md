### Subscriptions
```

```
GET    /monetization/royalties             - Get all royalties for user

### Pricing Recommendations

## Service Classes

### SubscriptionService
- `get_or_create(user_id)` - Get or initialize free subscription
- `upgrade(user_id, tier)` - Upgrade to paid tier with 14-day trial
- `cancel(user_id)` - Cancel subscription
- `TIER_FEATURES` - Configuration dict for each tier

### RoyaltyService
- `initialize_book(book_id, user_id)` - Initialize royalty tracking
- `record_sale(book_id, price_cents)` - Record book sale and calculate earnings
- `get_by_book(book_id)` - Get royalty record for book
- `get_user_royalties(user_id)` - Get all royalties for user

Payout Logic:
- Automatically triggers payout when earnings >= $500
- Payout scheduled 5 days after threshold crossing
- Platform takes 30% fee of royalties
- Author keeps 70%

### AffiliateService
- `create_link(user_id, book_id, source)` - Create new affiliate link
- `track_click(code)` - Record link click
- `record_conversion(code, revenue_cents)` - Record sale via affiliate
- `get_by_code(code)` - Get link stats
- `get_by_book(book_id)` - Get all links for book

### PricingService
- `generate_recommendation(book_id, current_price, genre)` - Generate AI recommendation
- `apply_recommendation(rec_id)` - Apply recommendation to book

Algorithm (Placeholder for ML):
- Books under $5: recommend 15% increase
- Books $5-10: recommend 5% increase
- Books over $10: recommend 5% decrease
- Confidence varies (0.70-0.80)

### CourseService
- `create_course(author_id, title, description)` - Create course
- `publish_course(course_id)` - Make course available
- `get_by_author(author_id)` - Get author's courses

### PatronService
- `create_account(creator_id, slug, tiers)` - Create patron account
- `add_patron(account_id, patron_id, tier_name)` - Add patron
- `get_by_creator(creator_id)` - Get creator's patron account

## Stripe Integration (TODO)

### Authentication Flow
1. GET /monetization/subscription/connect-stripe - Create Stripe connected account
2. User completes Stripe onboarding
3. Webhook: stripe.connect.account.updated
4. Save stripe_account_id to PatronAccount/AuthorSubscription

### Subscription Processing
1. POST /monetization/subscription/upgrade -> create stripe.Subscription
2. Webhook: customer.subscription.updated -> update subscription status
3. Webhook: customer.subscription.deleted -> cancel subscription

### Payout Processing
1. Daily job: check MarketplaceRoyalty.next_payout_at
2. Create payout via Stripe Connect
3. Update payouts_paid_cents, last_payout_at
