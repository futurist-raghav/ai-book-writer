### Workflow: Payout Processing

```
1. Author earns $500+ in royalties
   ↓
2. Payout threshold triggered (automatic via `record_sale`)
   ↓
3. Queue `process_payout_trigger` task
   ↓
4. Service updates RoyaltyRoyalty.payouts_pending
   ↓
5. Send `send_payout_initiated` email
   ↓
6. Send push notification to mobile app
   ↓
7. After 5 days: Bank transfer completes
   ↓
8. Webhook: Queue `complete_payout` task
   ↓
9. Update RoyaltyRoyalty.payouts_paid
   ↓
10. Send `send_payout_completed` email
```

### Workflow: Tier Upgrade

```
1. User clicks "Upgrade to Pro" button
   ↓
2. Frontend: GET /subscriptions/tiers → POST /subscriptions/upgrade
   ↓
3. Backend: Stripe creates subscription → Webhook confirms
   ↓
4. Queue `process_tier_upgrade` task
   ↓
5. Update AuthorSubscription tier + feature limits
   ↓
6. Send `send_tier_upgraded` email
   ↓
7. Push notification: "Welcome to Pro!"
   ↓
8. User sees new features unlocked
```

### Workflow: OAuth Token Refresh

```
Every 6 hours (via Celery Beat):
1. `refresh_expiring_oauth_tokens` task runs
   ↓
2. Find tokens expiring within 24 hours
   ↓
3. For each: Call `refresh_oauth_token` task
   ↓
4. OAuth service exchanges refresh token for new access token
   ↓
5. Update APIIntegration.access_token + token_expires_at
   ↓
6. If refresh fails: Queue `handle_token_refresh_error` task
   ↓
7. Send error notification to user

Daily at 2 AM (via Celery Beat):
1. `validate_active_integrations` task runs
   ↓
2. Verify all active tokens are still valid with provider
   ↓
3. Auto-refresh invalid tokens
   ↓
4. Mark permanently broken integrations as inactive
```

## Task Reference

### Payout Workflow Tasks

#### `process_payout_trigger(royalty_id)`

**Purpose:** Initiate a payout when royalties reach threshold

**Triggers:**
- Automatic: When `author_earnings_cents >= $500` (via `record_sale`)
- Manual: Admin triggers via API
- Scheduled: Via daily beat task

**Logic:**
1. Verify earnings above $500 minimum
2. Mark `payouts_pending_cents = author_earnings_cents`
3. Set `next_payout_at = now + 5 days`
4. Send `payout_initiated` email
5. Send push notification

**Retry Strategy:** 3 retries with 60s exponential backoff

```python
# Example: Triggered after book sale
royalty.author_earnings_cents = 550_00  # $550
# → process_payout_trigger.delay(royalty_id) automatically called
```

#### `complete_payout(royalty_id, transaction_id)`

**Purpose:** Mark payout as completed (called after bank transfer)

**Triggers:**
- Webhook from Stripe/bank
- Manual admin action
- API endpoint confirmation

**Logic:**
1. Move `payouts_pending_cents` → `payouts_paid_cents`
2. Update `last_payout_at`
3. Store `payout_reference` (transaction ID)
4. Send `payout_completed` email
5. Send push notification

```python
# Example: Bank transfer callback
complete_payout.delay(royalty_id, transaction_id="TXN-12345")
```

#### `handle_payout_failure(royalty_id, error_reason)`

**Purpose:** Handle failed payouts (invalid account, insufficient funds, etc.)

**Logic:**
1. Revert `payouts_pending_cents` to 0
2. Set `next_payout_at = now + 3 days` (retry)
3. Send `payout_failed` email with reason
4. Log error for admin review

```python
# Example: ACH transfer failed
handle_payout_failure.delay(
    royalty_id,
    error_reason="Bank account closed - update payment method"
)
```

#### `process_tier_upgrade(subscription_id, new_tier)`

**Purpose:** Complete tier upgrade including email and feature updates

**Logic:**
1. Get new tier's feature set
2. Update `AuthorSubscription.tier` + limits
3. Send `tier_upgraded` email
4. Send push notification
5. Trigger analytics event

```python
# Example: User upgrades to Studio
process_tier_upgrade.delay(subscription_id, "studio")
```

#### `process_tier_downgrade(subscription_id, new_tier)`

**Purpose:** Handle tier downgrade with data limit warnings

**Logic:**
1. Validate downgrade allowed
2. Update tier + limits
3. Warn if user exceeds new limits
4. Schedule downgrade completion

```python
# Example: User cancels Pro subscription
process_tier_downgrade.delay(subscription_id, "free")
```

### Periodic Beat Tasks

#### `process_pending_payouts()` - Daily 9 AM UTC

**Purpose:** Automatically process all payouts due today

**Logic:**
1. Query `MarketplaceRoyalty` where `next_payout_at <= today`
2. For each: Call `process_payout_trigger`
3. Log results

#### `send_earnings_milestones()` - Weekly (Monday 10 AM)

**Purpose:** Notify users when they reach earning milestones

**Milestones:** $100, $500, $1,000 (configurable)

**Logic:**
1. Find royalties in milestone windows
2. Check if already notified
3. Send email + push notification
4. Update notification flag

#### `send_earnings_summaries()` - Monthly (1st @ 8 AM)

**Purpose:** Send monthly earnings reports to all authors

**Contents:**
- Total earnings this month (gross + net)
- Books sold breakdown
- Payouts processed
- Next milestone progress
- Top performing books

## Email Templates

### Payout Initiated

**Subject:** "Your payout of $XXX has been initiated"

**Variables:**
- `user_name`: Author's display name
- `amount_cents`: Payout amount (e.g., 50000 → "$500.00")
- `book_title`: Top earning book
- `payout_date`: Date payout expected (e.g., "April 18, 2026")

**Sections:**
- Payout summary (amount, date)
- Payment method confirmation
- FAQ link
- Support contact

### Payout Completed

**Subject:** "Your payout of $XXX has been completed"

**Variables:**
- `amount_cents`: Payout amount
- `transaction_id`: Bank reference number
- `expected_arrival`: "1-3 business days"

**Sections:**
- Confirmation details
- Transaction ID for tracking
- Expected arrival (time frame)
- FAQ link

### Tier Upgraded

**Subject:** "Welcome to {{new_tier.upper()}}!"

**Variables:**
- `old_tier`: Previous tier (e.g., "free")
- `new_tier`: New tier (e.g., "pro")
- `new_features`: List of features (e.g., ["25 books", "5 collaborators"])

**Sections:**
- Welcome message
- Feature highlights
- Usage guide link
- Team contact

### Earnings Milestone

**Subject:** "Congratulations! You've earned ${{milestone}}! 🎉"

**Variables:**
- `milestone_amount_cents`: Milestone amount (e.g., 100_00 → "$100")
- `current_earnings_cents`: Total author earnings
- `book_title`: Top performing book

**Sections:**
- Milestone achievement
- Earnings summary
- Top book highlight
- Next milestone progress

## API Integration Points

### POST /books/{book_id}/sales

**Purpose:** Record a book sale (marketplace purchase)

```python
request_body = {
    "price_cents": 999,  # Price paid
    "platform": "marketplace",  # or "kdp", "draft2digital"
    "customer_id": "c_12345",  # Customer reference
}

# Backend automatically:
# 1. Calls RoyaltyService.record_sale()
# 2. Checks if payout threshold reached
# 3. Queues process_payout_trigger task if needed
```

### PATCH /subscriptions/{subscription_id}/tier

**Purpose:** Upgrade or downgrade subscription

```python
request_body = {
    "tier": "studio",  # New tier
    "billing_cycle": "annual",  # Or "monthly"
}

# Backend automatically:
# 1. Creates Stripe subscription
# 2. Queues process_tier_upgrade task
# 3. Triggers email + push notifications
```

### GET /integrations/{integration_id}/token-status

**Purpose:** Check if OAuth token is valid

**Response:**
```python
{
    "valid": True,  # or False
    "expires_at": "2026-04-20T10:30:00Z",
    "provider": "goodreads",
    "status": "active",  # or "refreshing", "invalid"
}
```

### POST /integrations/{integration_id}/refresh-token

**Purpose:** Manually trigger token refresh

```python
# Backend:
# 1. Calls refresh_oauth_token task
# 2. Returns updated token expiry
# 3. Or error if refresh fails
```

## Troubleshooting

### Payouts Not Processing

**Check:**
1. `celery -A app.tasks.celery_app inspect active` - workers alive?
2. `redis-cli info stats` - Redis OK?
3. Logs: `docker logs celery-worker`
4. Database: `SELECT * FROM marketplace_royalties WHERE next_payout_at <= NOW();`

**Common Issues:**
- Worker not running: Start with `celery ... worker`
- Redis down: Check `docker-compose ps`
- Task missing: Verify `app/workers/payout_workflow_tasks.py` imported in `celery_app.py`

### Emails Not Sending

**Check:**
1. SMTP credentials in `.env`
2. SendGrid API key valid
3. Logs: `app.services.email_notification_service`
4. Test: `python -c "from app.services.email_notification_service import EmailNotificationService; EmailNotificationService().send_payout_initiated(...)"`

### OAuth Tokens Not Refreshing

**Check:**
1. Task scheduled? Check `celery -A app.tasks.celery_app inspect scheduled`
2. Provider endpoints reachable: `curl https://www.goodreads.com/oauth/token -I`
3. Credentials stored? `SELECT * FROM api_integrations;`
4. Refresh token valid? Check expiry time
