## Overview

P7.8 introduces machine learning-powered content analysis features that leverage Claude AI to provide deep insights into writing quality, consistency, and effectiveness. This phase builds on existing AI integration to offer predictive, analytical, and coaching capabilities.

### Problem Statement

Writers need intelligent feedback beyond grammar/spell-check:
- **Consistency Checking**: Character voice changes, timeline inconsistencies
- **Pacing Analysis**: Identify slow sections, tension arcs
- **Style Insights**: Vocabulary richness, sentence complexity, readability
- **Content Quality**: Plot holes, cliché detection, emotional resonance
- **Audience Fit**: Genre conventions, reader expectations

## Core Features

### 1. Character Voice Analyzer

**Purpose**: Ensure character voices remain consistent throughout the manuscript

**Features:**
- **Voice Profile** (Auto-extracted from dialogue)
  - Vocabulary preferences (unique word choices)
  - Speech patterns (contractions, dialect, slang)
  - Sentence structure (short vs complex)
  - Emotional tone (positive, sarcastic, formal)

- **Consistency Tracking**
  - Flag dialogue that deviates from profile
  - Timeline of voice changes
  - Context for changes (character growth vs inconsistency)

- **Recommendations**
  - "This dialogue doesn't match [Character]'s voice. Suggestions:"
  - "Consider updating [Character]'s dialogue in chapter 3 to match voice evolution"

**UI Component**: `CharacterVoiceAnalyzer`
- Character selector dropdown
- Voice profile card (key traits)
- Deviation timeline
- Suggestion list with fixes

**Backend Tasks**:
- `@shared_task analyze_character_voices(book_id)`
- `@shared_task track_voice_changes(book_id, character_id)`
- Store results in `CharacterAnalysis` model

### 2. Timeline Validator

**Purpose**: Detect temporal inconsistencies and continuity errors

**Features:**
- **Event Extraction**
  - Parse chapter dates and events
  - Timeline reconstruction
  - Age/timeline consistency checks

- **Conflict Detection**
  - Flagging impossible sequences (death before birth, age jumping)
  - Gap identification (missing time markers)
  - Geographic impossibilities (travel time violations)

- **Visual Timeline**
  - Gantt-style chart of events
  - Character age progression
  - Multi-character timeline view

**UI Component**: `TimelineValidator`
- Interactive timeline chart
- Conflict list with severity
- Character age tracker
- Suggestion recommendations

**Example Issues:**
- "Character age is 25 in Ch. 3 but 20 in Ch. 8 (7 chapters later)"
- "Character dies in Ch. 12 but reappears mourned in Ch. 15"
- "Travel from NY to LA in 2 hours (impossible, takes 5+ hours)"

### 3. Pacing Analyzer

**Purpose**: Analyze narrative rhythm and tension curves

**Features:**
- **Pacing Metrics**
  - Average scene length
  - Chapter length distribution
  - Action vs dialogue vs description ratios
  - Tension curve mapping

- **Slow Section Detection**
  - Identify chapters with low action density
  - Flag lengthy descriptions without narrative movement
  - Suggest pacing improvements

- **Emotional Arc Analysis**
  - Map emotional intensity across chapters
  - Identify climax and turning points
  - Validate act structure (3-act or other)

- **Reader Focus Predictor**
  - Estimate reader engagement intensity
  - Predict dropout points
  - Suggest pacing adjustments

**UI Component**: `PacingAnalyzer`
- Multi-line chart (action, dialogue, description, intensity)
- Chapter length heatmap
- Emotional arc visualization
- Slow section flagging

**Metrics:**
- Scene length (avg: 3,000-4,000 words)
- Action ratio (optimal: 20-40% per chapter)
- Description ratio (optimal: 10-25% per chapter)
- Intensity score (0-100 per scene)

### 4. Style Analyzer

**Purpose**: Measure and improve prose quality and consistency

**Features:**
- **Vocabulary Metrics**
  - Unique word count (richness)
  - Average word length
  - Vocabulary complexity (Flesch-Kincaid grade)
  - Key terms frequency

- **Sentence Analysis**
  - Average sentence length
  - Sentence variety (simple/complex ratio)
  - Fragment usage
  - Parallelism scoring

- **Readability Scoring**
  - Flesch Reading Ease (0-100)
  - Gunning Fog Index
  - SMOG index
  - Target audience alignment

- **Tone Consistency**
  - Author voice fingerprinting
  - POV consistency checking
  - Tense consistency (present vs past)
  - Perspective shifts flagging

- **Prose Quality**
  - Show vs tell ratio
  - Passive voice percentage (flag excessive)
  - Adverb density (flag overuse)
  - Cliché phrase detection

**UI Component**: `StyleAnalyzer`
- Readability gauge (target: 60+)
- Vocabulary richness chart
- Sentence variety breakdown
- Tone consistency heatmap
- Problem area highlighting (passive, clichés, etc.)

**Benchmarks:**
- Genre-specific targets (literary: 50-60, thriller: 60-70, YA: 70+)
- Vocabulary richness: 8,000+ unique words for 80k+ manuscript
- Sentence variety: 20% simple, 50% complex, 30% compound

### 5. Plot Hole Detector

**Purpose**: Identify logical inconsistencies and narrative gaps

**Features:**
- **Story Logic Validation**
  - Premise consistency checking
  - Character motivation validation
  - Consequence tracking (actions should have results)
  - Setup-payoff matching

- **Plot Gap Detection**
  - Missing explanations for major events
  - Unanswered questions
  - Character knowledge gaps (characters shouldn't know unrevealed info)
  - Unresolved threads

- **Causality Checking**
  - "This happens, but why?" detection
  - Missing causal links between events
  - Probability issues (unrealistic coincidences)

- **Genre-Specific Logic**
  - Mystery: All clues must allow solution
  - Romance: Relationship progression must feel earned
  - Thriller: Pacing and stakes escalation
  - Fantasy: Magic system consistency

**UI Component**: `PlotHoleDetector`
- Issue severity ranking (critical/major/minor)
- Issue type categorization
- Affected chapters
- Suggested fixes

**Example Issues:**
- Critical: "Character uses ability introduced in Ch. 8 to solve Ch. 3 problem"
- Major: "Why did Character X agree to the plan? No motivation established"
- Minor: "Minor character acts knowledgeable about secret, but wasn't present when revealed"

### 6. Cliché Analyzer

**Purpose**: Identify overused tropes and suggest fresh alternatives

**Features:**
- **Cliché Detection**
  - Common phrases (tired metaphors, overused descriptions)
  - Trope identification (chosen one, love triangle, etc.)
  - Dialogue clichés (don't say X, show X)
  - Scene clichés (chase scene, betrayal reveal, etc.)

- **Trope Distribution**
  - All tropes used in manuscript
  - Frequency analysis
  - Genre-appropriate assessment (some tropes are fine in romance)
  - Originality scoring

- **Fresh Alternative Suggestions**
  - Specific rewrites for clichés
  - Uncommon alternatives for common tropes
  - Genre-aware suggestions

- **Authenticity Scoring**
  - "How original is this manuscript?" (1-10)
  - Comparison to similar published books
  - Unique elements identified

**UI Component**: `ClichéAnalyzer`
- Cliché hit list with severity
- Rewrite suggestions with before/after
- Trope distribution pie chart
- Alternative suggestions
- Originality score

**Example:**
- Phrase: "Her heart pounded in her chest" → Cliché
  - Suggestions: "Her heartbeat thundered" / "Her pulse quickened"
- Trope: "Love triangle in paranormal romance" → Common (acceptable)
- Scene: "Betrayal at climax" → Cliché trope, less severe than phrase

## API Endpoints

### Analysis Triggers

```python
# Request full analysis
POST /api/v1/books/{book_id}/analyze
{
    "analysis_types": ["character_voice", "timeline", "pacing", "style", "plot", "cliche"],
    "chapters": [1, 2, 3],  # Optional: specific chapters
    "depth": "quick" | "thorough"  # Quick: 2 min, Thorough: 10 min
}

# Response (async, returns job_id)
{
    "job_id": "job_123456",
    "status": "processing",
    "estimated_completion": "2026-04-13T10:30:00Z"
}

# Check analysis status
GET /api/v1/books/{book_id}/analyses/{job_id}

# Get specific analysis results
GET /api/v1/books/{book_id}/analyses/character-voice?character_id=char_123
GET /api/v1/books/{book_id}/analyses/pacing
GET /api/v1/books/{book_id}/analyses/style
GET /api/v1/books/{book_id}/analyses/plot-holes
GET /api/v1/books/{book_id}/analyses/cliches

# List analysis history
GET /api/v1/books/{book_id}/analyses?limit=10&offset=0
```
