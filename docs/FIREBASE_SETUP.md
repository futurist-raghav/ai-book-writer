# Firebase Cloud Messaging Setup for Expo

This guide covers setting up Firebase Cloud Messaging (FCM) with Expo for A Book Writer mobile app notifications.

## For Development/Testing

**No Firebase configuration needed!** Expo provides a built-in push notification service that handles all the complexity:
- Device tokens are generated by Expo
- Push service handles delivery to FCM (Android) and APNs (iOS)
- App can send/receive notifications without Firebase credentials

To test notifications in development:
```bash
# 1. Start the mobile app (it auto-registers device token)
npm --prefix mobile run ios  # or android

# 2. Run E2E tests
python scripts/test_notifications_e2e.py --api-url http://localhost:8000 --token YOUR_AUTH_TOKEN

# 3. Send test notification
curl -X POST http://localhost:8000/api/v1/notifications/send-test \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

## For Production Build

### Step 1: Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Click "Create Project" and follow the wizard
3. Project name: "AI Book Writer" (or your preference)
4. Select "Create project" (default settings are fine)

### Step 2: Add Android App

1. In Firebase Console, click the Android icon to add an Android app
2. **Package name**: `com.aiebookwriter.app` (update with your actual package)
3. **App nickname**: `AI Book Writer Android`
4. Click "Register app"
5. Download the `google-services.json` file
6. Store safely (don't commit to git)

### Step 3: Add iOS App

1. In Firebase Console, click the iOS icon
2. **Bundle ID**: `com.aiebookwriter.app` (must match your Expo config)
3. **App nickname**: `AI Book Writer iOS`
4. Click "Register app"
5. Download the `GoogleService-Info.plist` file
6. Store safely (don't commit to git)

### Step 4: Configure Expo EAS

```bash
# Initialize EAS if not already done
eas init

# Create credentials for iOS
eas credentials

# Create credentials for Android
eas credentials
```

### Step 5: Update `eas.json`

```json
{
  "build": {
    "development": {
      "distribution": "internal",
      "android": {
        "serviceAccountKeyPath": "./secrets/firebase-key.json"
      },
      "ios": {
        "enterpriseRegistrationEnabled": true
      }
    },
    "preview": {
      "distribution": "internal",
      "android": {
        "serviceAccountKeyPath": "./secrets/firebase-key.json"
      }
    },
    "production": {
      "android": {
        "serviceAccountKeyPath": "./secrets/firebase-key.json"
      }
    }
  },
  "submit": {
    "production": {
      "android": {
        "track": "production"
      },
      "ios": {}
    }
  }
}
```

### Step 6: Store Firebase Credentials Securely

For Expo EAS builds, store credentials as secrets:

```bash
# Create a secret for the Firebase service account key
eas secret:create FIREBASE_SERVICE_ACCOUNT_KEY

# Paste the JSON content from google-services.json (Android)

# For iOS, upload credentials through EAS dashboard
```

### Step 7: Build Production App

```bash
# Build for iOS
eas build --platform ios --auto-submit

# Build for Android
eas build --platform android

# Or build both
eas build --platform all
```

## Environment Variables

Update your `.env.production` with Firebase project details:

```env
# Firebase Project Configuration
EXPO_PUBLIC_FIREBASE_API_KEY=xxx
EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=xxx.firebaseapp.com
EXPO_PUBLIC_FIREBASE_PROJECT_ID=xxx
EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=xxx.appspot.com
EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=xxx
EXPO_PUBLIC_FIREBASE_APP_ID=xxx
```

These should match your Firebase project settings.

## Debugging Firebase Issues

### No notifications received?

1. **Verify Expo token registration:**
   ```bash
   # Check if device is registered
   curl http://localhost:8000/api/v1/notifications/unread/count \
     -H "Authorization: Bearer YOUR_TOKEN"
   ```

2. **Check Firebase service account:**
   ```bash
   # Verify JSON has required fields:
   # - type: "service_account"
   # - project_id
   # - private_key_id
   # - private_key
   # - client_email
   ```

3. **Enable FCM API in Google Cloud:**
   - Go to Google Cloud Console
   - Select your Firebase project
   - Search for "Firebase Cloud Messaging API"
   - Click "Enable"

4. **Check app signatures for Android:**
   - Firebase requires correct app signing key
   - Verify with: `eas credentials show`

### Check Expo Logs
```bash
# View device logs while testing
eas build-logs --interactive
```

### Verify Backend Sending
```bash
# Check if backend actually sent the notification
docker compose logs backend | grep notification
```

## Troubleshooting

### "Failed to create credentials"
- Ensure you have an Expo account
- Verify you're logged in: `eas login`
- Check EAS is initialized: `eas init`

### "Service account key invalid"
- Re-download the JSON from Firebase
- Ensure no extra whitespace or formatting issues
- Verify the key has `"type": "service_account"`

### "Signing key not found" (Android)
- Run `eas credentials` and select "Set progress"
- Expo will guide you through signing key setup
- For production, use an existing key if available

### iOS certificates expired
- Go to Apple Developer
- Renew certificates and profiles
- Update credentials: `eas credentials delete --platform ios`
- Re-upload new certificates

## Security Best Practices

1. **Never commit Firebase keys to git**
   - Add `secrets/` to `.gitignore`
   - Store in EAS secrets instead

2. **Use environment-specific keys**
   - Development: Expo built-in service
   - Staging: Testing Firebase project
   - Production: Production Firebase project

3. **Rotate credentials regularly**
   - Every 90 days for service accounts
   - Update EAS secrets after rotation

4. **Restrict Firebase permissions**
   - Service account should only have FCM permissions
   - Use IAM roles to limit access

## Advanced: Direct Firebase Integration

For more control over notifications, you can integrate Firebase SDK directly:

```bash
npm install firebase-messaging-expo
```

Then in your notification service:
```typescript
import { getMessaging, onMessage } from 'firebase/messaging/compat';

const messaging = getMessaging();
onMessage(messaging, (payload) => {
  // Handle foreground notifications
});
```

However, this is optional - **Expo's push service handles this transparently**.
