## Key Features

### 1. Team Management
- **Hierarchical Organization:** Teams contain members with roles
- **Invitation Flow:** Invite by email, track acceptance
- **Soft Deletion:** Deactivate without data loss
- **Subscription Limits:** max_members, max_books, max_api_keys per tier

### 2. Role-Based Access Control (RBAC)
- **Built-in Roles:** Owner, Lead, Member, Guest
- **Custom Roles:** Define fine-grained permissions per resource type
- **Composition:** Resources like "books" support ["view", "edit", "delete"] actions
- **Permission Levels:** None → View → Comment → Edit → Delete → Admin

### 3. Single Sign-On (SSO)
- **Providers:** Okta, Azure AD, Google OAuth2, SAML 2.0
- **Auto-Provisioning:** Automatically create users on first login
- **Domain Enforcement:** Restrict to @company.com emails
- **MFA Support:** Enforce multi-factor authentication per team
- **Group Sync:** Sync LDAP/AD groups to team roles (flag: sync_groups)

### 4. Comprehensive Audit Logging
- **Immutable Records:** All events appended, never modified
- **Event Types:** User actions, permission changes, resource operations
- **Full Context:** Actor, IP address, user agent, old/new values
- **Compliance Ready:** Supports GDPR/HIPAA audit requirements
- **Resource History:** Full change timeline for any resource

### 5. Fine-Grained Permissions
- **Resource-Level Control:** Override role permissions per book/chapter
- **Conditional Access:** Time-limited permissions with expiry
- **Hierarchical Checks:** Permission level >= required level
- **Efficiency:** Cached permission checks to avoid N+1 queries

### 6. Activity Analytics
- **Daily Aggregation:** Track edits, comments, creations, AI usage
- **Team Insights:** Active members, feature usage trends
- **Billing Integration:** Usage tracking for usage-based tiers
- **Dashboard Ready:** Time-series data for analytics UI

## Security Considerations

1. **Client Secrets:** Store encrypted in database (use `cryptography` library)
2. **Audit Logs:** Immutable (no DELETE permissions, append-only)
3. **Permission Checks:** Always re-verify before resource access (no client-side trust)
4. **SSO Signatures:** Validate SAML/OAuth signatures before accepting
5. **Rate Limiting:** Enforce login attempts, API key usage per tier
6. **IP Whitelisting:** Optional per-team IP restriction (P7.7.3)
