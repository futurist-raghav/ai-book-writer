## Why Docs Were Archived

Many files in this repository were historical snapshots of the same topic (especially deployment status reports and session summaries). That created 3-4 variants for a single concern and made it hard to know which file is the source of truth.

To reduce ambiguity:

- Canonical docs remain in this root folder.
- Historical reports and session logs were moved to `archive/legacy/`.
- Active docs should be updated in place instead of creating new "_COMPLETE", "_FINAL", "_STATUS" variants.

## Archive

- `archive/legacy/` - Historical reports, completion snapshots, and superseded docs kept for reference.
