## Webhook Events

### Available Events
```
manuscript.published
  - Fired when book is published
  - Data: book_id, title, platforms, isbn

chapter.completed
  - Fired when chapter marked done
  - Data: book_id, chapter_id, word_count

feedback.received
  - Fired when feedback added
  - Data: book_id, feedback_type, from_user

royalty.earned
  - Fired when royalty processed
  - Data: book_id, amount_cents, platform

collaboration.invited
  - Fired when collaborator invited
  - Data: book_id, role, invitee_email
```

### Webhook Retry Logic
- Initial: Immediate delivery
- Fail: Exponential backoff (5s, 30s, 2m, 10m, 1h)
- Max retries: 5
- Total window: ~1.5 hours

## Integration Flows

### Zapier Integration Flow
1. User clicks "Connect to Zapier"
2. OAuth authorization to Zapier
3. IntegrationAuth created with OAuth tokens
4. User builds Zap in Zapier UI
5. Zap can trigger actions: create_chapter, add_comment, update_event
6. ZapierAction logged and executed
7. Result stored in ZapierAction.result

### Notion Sync Flow
1. User connects Notion (OAuth)
2. Selects target database
3. NotionSync created with mapping config
4. Daily job queries books (filter by last_sync_at)
5. Creates/updates Notion rows via Notion API
6. Bidirectional: Notion changes synced back to Scribe House
7. Sync status stored in last_sync_at + synced_records_count

### Google Docs Import Flow
1. User connects Google Drive (OAuth)
2. Selects document
3. GoogleDocsSync created
4. Initial import: fetch full document, create/update chapter
5. Ongoing: watch for document changes
6. On change: fetch delta, merge with existing content
7. Last edit time tracked to avoid conflicts

### Webhook Flow (Example: Book Published)
1. Book published (status = "published")
2. trigger_event("manuscript.published", {book_id, title, platforms})
3. Query webhooks subscription for "manuscript.published"
4. Create WebhookDelivery records
5. Async job POSTs to webhook URLs in parallel
6. Store response status, body, time
7. On failure: schedule retry with exponential backoff

## OAuth Provider Configuration

### Zapier
- Redirect URI: `https://api.aibook.com/auth/zapier/callback`

- Client ID/Secret: From Zapier Platform

### Notion
- Redirect URI: `https://api.aibook.com/auth/notion/callback`
- Scopes: `read`, `write` on databases
- Client ID/Secret: From Notion Integrations

### Google Drive/Docs
- Redirect URI: `https://api.aibook.com/auth/google/callback`
- Scopes: `drive`, `docs` read/write
- Client ID/Secret: From Google Cloud Console

### Slack
- Redirect URI: `https://api.aibook.com/auth/slack/callback`
- Scopes: `chat:write`, `files:read`
- Client ID/Secret: From Slack App Settings

## Async Jobs Required

1. **Webhook Delivery Job** - Every 30s
   - Query pending WebhookDelivery records
   - POST to webhook.url with payload
   - Update status, response_body, response_time_ms
   - Schedule retry if failed

2. **Integration Sync Job** - Every 60m (configurable)
   - Query IntegrationAuth with sync_enabled=1
   - For each: call provider sync (notion, google_docs, etc)
   - Update last_sync_at, error tracking
   - Log success/failure

3. **OAuth Refresh Job** - Hourly
   - Query IntegrationAuth with oauth_refresh_token
   - Check if expires_at < now + 1 hour
   - Refresh token via provider API
   - Update oauth_access_token, oauth_expires_at

4. **Export Template Job** - Every 5m
   - Query ExportTemplate with webhook_trigger_enabled=1
   - Check if recent events match trigger_events
   - Execute export (query data per template config)
   - POST to webhook if configured
