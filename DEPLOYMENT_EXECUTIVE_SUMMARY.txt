DEPLOYMENT PROCEDURE (15 minutes)
═════════════════════════════════════════════════════════════════════════════════

STEP 1: Initialize Database (5 minutes)
  
  SSH to VM:
  $ gcloud compute ssh scribe-house --zone=asia-south1-c

  Initialize database:
  $ cd ~/backend
  $ python init_db.py
  
  Expected result:
  ✅ Database tables created successfully!


STEP 2: Start Backend Service (2 minutes)
  
  Restart systemd service:
  $ sudo systemctl restart aiwriter-backend
  
  Verify status:
  $ sudo systemctl status aiwriter-backend
  
  Expected result:
  ● aiwriter-backend.service - Loaded


STEP 3: Health Check (1 minute)
  
  Test backend health:
  $ curl http://34.14.172.251:8000/health
  
  Expected result:
  {"status": "healthy", "version": "1.0.0"}


STEP 4: Deploy Cloudflare Worker (2 minutes)
  
  Deploy frontend:
  $ cd ~/Projects/AI-Book-Writer/frontend
  $ wrangler deploy
  
  Expected result:
  ✓ Your worker has been published to:
    https://scribe-house-prod.raghav-10168-19.workers.dev


STEP 5: End-to-End Test (5 minutes)
  
  Test through Cloudflare:
  $ curl https://scribe-house-prod.raghav-10168-19.workers.dev/health
  
  Test direct to backend:
  $ curl http://34.14.172.251:8000/api/v1/health
  
  Expected result:
  Both return {"status": "healthy", "version": "1.0.0"}

RISK MITIGATION
═════════════════════════════════════════════════════════════════════════════════

Potential Issues & Mitigation:

1. Database Schema Creation
   • Risk: Schema doesn't match models
   • Mitigation: init_db.py validates all tables created
   • Fallback: Snapshot schema from development environment

2. STT Model Download
   • Risk: Gemma 4 download takes > 15 minutes
   • Mitigation: 15-minute timeout, graceful fallback to Whisper
   • Fallback: Whisper API on VM or OpenAI

3. Ollama Service
   • Risk: Ollama not installed or not running
   • Mitigation: OllamaInitializer detects and auto-starts
   • Fallback: Backend continues with Whisper API

4. Cloudflare Routing
   • Risk: Domain/subdomain not pointing correctly
   • Mitigation: Tested via wrangler deploy
   • Fallback: Access directly at http://34.14.172.251:8000

5. Network/Firewall
   • Risk: Port 8000 not accessible
   • Mitigation: VM security group checked
   • Fallback: Cloudflare Worker can proxy
